# Canopy · Nature-Tech Trading Terminal

轻量化自动化加密货币量化交易终端。策略层内置 5 种策略 + 策略工厂，回测引擎含完整绩效指标，风险控制器 6 步审批链，订单执行器通过 CCXT 对接交易所。

## 架构

```
canopy/
├── main.py                  # 主入口（pywebview + CanopyAPI 16 方法）
├── config.py                # 配置管理
├── engine/                  # 策略引擎
│   ├── base.py              # Strategy 抽象基类
│   ├── factory.py           # 策略工厂 + 自定义组合
│   ├── trend.py             # 趋势跟踪（双 EMA + ATR 止损）
│   ├── grid.py              # 网格交易（等差/等比）
│   ├── arbitrage.py         # 跨所套利
│   ├── momentum.py          # 动量突破（Donchian + 成交量）
│   ├── mean_reversion.py    # 均值回归（Z-Score + 布林带）
│   ├── runner.py            # 策略运行器（多线程 + 信号分发）
│   ├── risk.py              # 风控控制器（6 步审批链 + 熔断器）
│   ├── executor.py          # 订单执行器（队列 + CCXT 下单）
│   ├── backtest_runner.py   # 回测运行器（统一 API）
│   ├── fallback.py          # 模拟数据生成（GBM）
│   └── backtest/            # 回测引擎
│       ├── engine.py        # 事件驱动回测循环
│       └── metrics.py       # 绩效指标（Sharpe/Sortino/Calmar）
├── exchange/
│   └── ccxt_adapter.py      # CCXT 交易所适配器（12 方法）
├── data/
│   └── fetcher.py           # 行情数据层（Parquet 缓存）
└── web/
    └── index.html           # Nature-Tech Glass 前端（4 主题 × 2 模式）
```

## 信号 → 交易完整链路

```
策略 on_bar() 产生信号
  → RiskManager.approve() 6 步审批
    → OrderExecutor.submit() 入队
      → CCXT create_order() 下单
        → 成交回调 → 更新 RiskManager 持仓
```

## 风控审批链

1. 熔断器检查
2. 每日亏损限额（5%）
3. 全局回撤熔断（15%）
4. 信号有效性
5. 仓位限制（单笔 ≤ 5%）
6. 敞口限制（总敞口 ≤ 80%）

## 启动

```bash
pip install -r requirements.txt
python canopy/main.py
```

## 技术栈

- **后端**: Python 3.10+, CCXT, pandas, numpy
- **前端**: pywebview + 原生 HTML/CSS/JS（Canvas 净值曲线）
- **风格**: Nature-Tech Glass · 4 色彩主题 × 暗/亮模式
